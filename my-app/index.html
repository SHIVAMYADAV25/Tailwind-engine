Why Chai

your own terms.

Every class uses your custom prefix — chai-, ui-, app- — so it never clashes with existing stylesheets.

⚡

Build-time generation

Run npm run build and Chai scans your HTML, collects every class, and outputs a single minimal CSS file.

🪶

Zero runtime overhead

How it works

From class name
to CSS in 3 steps.

Chai's engine parses your HTML at build time, resolves every utility class, and emits a minimal stylesheet. No surprises.

01

Scaffold your project

Run the CLI to create a new app with a ready-made template and build script.

npx create-chai-app my-app
02

Write utility classes

Use chai-* classes directly in your HTML. Responsive, hover, and state variants all work.

<div class="chai-p-8 chai-bg-blue-600 chai-hover-scale-105">
03

Build & ship

Run the build command. Chai scans your files, generates a single optimized chai.css, and you're done.

npm run build → dist/chai.css ✓
